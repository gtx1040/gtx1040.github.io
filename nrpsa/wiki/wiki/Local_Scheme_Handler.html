Local Scheme Handler

From Nexus RP: Wiki

Jump to: navigation, search

In many cases it is necessary to access files stored on the local disk from browsers.

FROM VERSION 1.5.1 r7439 ONWARDS

Scheme format

http://mta/resourceName/file.extension
  • resourceName: The resource name, local is a special name for the current resource
  • file.extension: The file path within resourceName

You can use this scheme format for loadBrowserURL as well as in HTML files e.g. <img> tags.


Minimum supported server 1.5.0-9.07439
Minimum supported client n/a
Note: Using this function requires the resource to have the above minimum version declared in the meta.xml <min_mta_version> section. e.g. <min_mta_version server="1.5.0-9.07439" />

Examples

Example 1

http://mta/myResource/assets/myImage.png

Refers to the file assets/myImage.png in a resource named myResource

Example 2

http://mta/local/myImage.png

Refers to the file myImage.png in the current resource

BEFORE VERSION 1.5.0-7439

Before build 1.5.0-7439 local files were accessed using the nrplocal:// scheme. This scheme has been deprecated now, but throws a warning and should not be used any longer for this reason. Nevertheless, it still works for backward compatibility, but should be replaced as soon as possible. Don't forget to set min_mta_version though (when using the new scheme).
