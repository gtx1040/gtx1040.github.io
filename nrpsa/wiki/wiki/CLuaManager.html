Function definition mapping

These are for the function definitions, and where certain things are placed. As a rule of thumb if a function takes a player for its first argument you'll find it in the player definitions file (adapt this for whatever class you want). For now, this stuff reflects my (qaisjp's) local repository - there is one on GitHub here.

A quick note about this branch: it's a huge bunch of changes so I progressively rebase back onto master so that I don't have to deal with a huge merge conflict when I am ready to merge this to the master. This means that if you pull this, the next time you want to pull you will have a completely different repo (from the point of my changes) with the same changes, meaning that any of your changes will break.

Until this is merged into master, this section will serve as a todo/progress list.

Backwards compatibility functions remain in "CLuaFunctionDefs.h".

Things with "todo" at the beginning aren't currently done, but are expected to be completed.
