Before you start

First of all, please ensure your computer fits the requirements needed. Read the Client Manual page for further informations, or join us on IRC @ irc://irc.nexusrp.com/nrp

Requirements

The hardware requirements for Nexus RP: San Andreas are the same than on Windows. For software requirements, you need:

  • Wine (get it on your package manager (synaptic, apt, pacman, yum etc.); follow instructions here if you're using Ubuntu

(as of NRP 1.4.1 Ubuntu's default Wine package seems to be incompatible with Visual C++, which is required to install NRP)

  • Windows fonts installed:
    • tahoma.ttf
    • tahomabd.ttf
    • verdana.ttf

Get them on the Internet, e.g. fontonic.com

Or get the zip archive with the 3 fonts here: wine_nrp_fonts.zip

Install them in: 
~/.wine/dosdevices/c:/windows/Fonts/

(~/ points to your user home folder on Linux, .wine is the default wineprefix)

Installing the game

Download the NRP installer from nrp.us.to, ensure you can launch it (chmod +x) then install it when you want.

Running the game

Once installed, double-clicking on the Nexus RP.exe should work. If not, try in a terminal the following command: "wine Nexus RP.exe" in the directory you installed Nexus RP to. If it doesn't work, check the contents of the file c:/Program_Files/Nexus RP/NRP/CEGUI.log, which may contain useful information.

Known issues

Specific issues

  • Impossible connection through the server browser [Fixed in the 5084 bug ]
  • NRP won't start on Ubuntu 12.04 LTS [Temporary fix is available here ]

Other issues

  • NRP isn't starting (even with fonts installed)
  1. Try to start NRP:SA in a virtual desktop
    Go to WineConfig, choose the tab "Graphics" and select "Emulate a virtual desktop".
  2. Try to delete your gta_sa.set file
    which is located in the "GTA San Andreas User Files" folder, which can be found in your home directory.
    (Remember to create a copy, if you're playing San Andreas in singleplayer)
  3. Try to delete your NRP config file
    which is: "NRP San Andreas 1.3/NRP/coreconfig.xml"
    (Also remember to create a copy, if you don't want to lose your edited NRP configuration)
  • Crash when connecting
    Sometimes the audio-server makes problems (could be related to pulseaudio), in this case you've to go to WineConfig and choose the tab Audio, then deselect "ALSA" and select "EsoundD". Save the settings and restart NRP.
  • Special Detections (SD)
    If you are using a 64 wine version you may have problems with Special Detections. If the server you are trying to connect keeps showing something like this, then you should make a 32 bit wine prefix (or bottle). 
export WINEARCH=win32 WINEPREFIX=~/.winegta
winecfg

Change Windows version on bottom to Windows 7 and press OK. Now you have a 32 bit wine prefix on ~/.winegta. Install GTA:SA and them NRP. After this, NRP and GTA have been installed within ~/.winegta prefix which is a 32 bit wine environment.
